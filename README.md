## ⚡ Quick Start

### Prerequisites
- Node.js v18+ 
- npm v9+

### 1. Start the Backend Server

```bash
cd server
npm install
npm run prisma:migrate   # Creates SQLite dev.db
npm run db:seed          # Populates demo data & users
npm run dev              # Starts on port 5000
```

### 2. Start the Frontend

```bash
cd client
npm install
npm run dev              # Starts on port 5173
```

Then visit **http://localhost:5173** in your browser.

## 🤖 AI Detection Engines

The platform uses a **pluggable engine architecture** — no frontend changes needed to switch providers:

| Engine | When Active | Capabilities |
|--------|-------------|--------------|
| **Mock/Statistical** | Always available (default fallback) | True Perplexity + Burstiness math for text; metadata/noise analysis for images; temporal mock for video |
| **Google Gemini** | Set `GEMINI_API_KEY` | Multimodal image vision + language analysis |
| **Hugging Face** | Set `HUGGING_FACE_API_KEY` | RoBERTa text classifier + ViT image detector |

### Switching Engines
- **Admin Panel → API Keys tab**: Enter keys live — no server restart needed.
- **Per-scan selector**: Each scan page has an engine dropdown.

## 📡 API Reference

### Authentication
| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/register` | Create account → sends OTP |
| POST | `/api/auth/verify-otp` | Verify email OTP → returns JWT |
| POST | `/api/auth/login` | Sign in → returns JWT |
| POST | `/api/auth/forgot-password` | Request password reset OTP |
| POST | `/api/auth/reset-password` | Reset with OTP + new password |
| GET | `/api/auth/profile` | Get authenticated profile |
| PUT | `/api/auth/profile` | Update name / email |
| PUT | `/api/auth/change-password` | Change password |
| DELETE | `/api/auth/delete-account` | Permanently delete account |

### Detection
| Method | Endpoint | Body / File |
|--------|----------|-------------|
| POST | `/api/detect/image` | `multipart/form-data` – file (PNG/JPG/WEBP) + provider |
| POST | `/api/detect/video` | `multipart/form-data` – file (MP4/MOV/AVI) + provider |
| POST | `/api/detect/text` | JSON `{ text, provider? }` |

### History & Dashboard
| Method | Endpoint | Query Params |
|--------|----------|--------------|
| GET | `/api/history` | `page`, `limit`, `search`, `fileType`, `result` |
| GET | `/api/history/dashboard` | Returns stats + weekly chart data |
| DELETE | `/api/history/:id` | Delete one scan entry |

### Admin (ADMIN role required)
| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/admin/stats` | Server metrics, user counts, API usage |
| GET | `/api/admin/users` | All users with scan counts |
| DELETE | `/api/admin/users/:id` | Delete user + cascade data |
| GET | `/api/admin/feedback` | All feedback submissions |
| GET | `/api/admin/audit` | Latest 200 security audit logs |
| PUT | `/api/admin/settings` | Update AI provider API keys live |

## 🗄️ Database Schema (Prisma / SQLite)

```
User ──────────────┐
  id, email, name  │ 1:N
  passwordHash     │──── DetectionHistory ─── ImageResult
  role, isVerified │                       ├── VideoResult
  createdAt        │                       └── TextResult
                   │
                   ├── Otp (verification codes)
                   ├── Notification
                   ├── Feedback
                   └── AuditLog

APIUsage (standalone provider call logging)
```

**Switch to Supabase/MySQL**: Change `DATABASE_URL` in `.env` and run `npm run prisma:migrate`.

## 🔒 Security Features

- **JWT authentication** with 15-minute access tokens + 7-day refresh tokens
- **bcrypt** password hashing (salt rounds: 10)
- **Helmet** HTTP security headers
- **CORS** restricted to configured frontend origin
- **Rate limiting**: 10 auth requests / 30 scans per 15 minutes
- **Zod validation** on all request bodies
- **Audit logging** for all auth and admin events
- **SQL injection protection** via Prisma ORM parameterized queries
